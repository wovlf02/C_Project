## 1. 모듈 구조

```
device/
├── src/
│   ├── main.c              # 진입점, 초기화, 메인 루프
│   ├── network.c           # TCP 연결, 메시지 송수신, 하트비트
│   ├── protocol.c          # JSON 메시지 빌드/파싱 (서버와 공유 가능)
│   ├── display.c           # ncurses UI 렌더링
│   ├── input.c             # 사용자 입력 처리 (전화번호, 메뉴)
│   ├── sensor.c            # 센서 데이터 시뮬레이션
│   ├── storage.c           # Encrypted Storage (로컬 암호화 저장)
│   └── config.c            # 기기 설정 (서버 IP, 포트, device_id)
├── include/
│   ├── device.h            # 기기 상태 구조체
│   ├── network.h           # 네트워크 함수 선언
│   ├── protocol.h          # 프로토콜 정의 (서버와 동일)
│   ├── display.h           # 화면 함수 선언
│   ├── sensor.h            # 센서 함수 선언
│   └── storage.h           # 암호화 저장소 함수 선언
└── Makefile
```

## 3. 센서 시뮬레이션 (sensor.c)

### 3.1 시뮬레이션 모드

| 모드 | 설명 |
|------|------|
| NORMAL | RPM/속도가 일정 범위 내에서 랜덤 변동 |
| INCREASE | 점차 강도 증가 (워밍업 시뮬레이션) |
| DECREASE | 점차 강도 감소 (쿨다운 시뮬레이션) |
| IDLE | 센서 값 0 (비활동 상태 — 자동 로그아웃 테스트용) |

### 3.2 센서 데이터 구조

```c
typedef struct {
    int rpm;           // 분당 회전수 (0~200)
    float speed;       // 속도 km/h (0~30)
    int resistance;    // 저항 레벨 (1~20)
} SensorData;
```

### 3.3 기구 유형별 센서 프리셋

| 기구 유형 | RPM 범위 | 속도 범위 | 저항 범위 |
|-----------|----------|-----------|-----------|
| cardio (트레드밀) | 0~180 | 0~20 | 1~15 |
| cardio (자전거) | 0~120 | 0~40 | 1~20 |
| strength | 0~30 | - | 1~20 |
| stretching | 0~10 | - | - |

## 4. Encrypted Storage (storage.c)

### 4.1 역할

- 운동 중 세션 데이터를 기기 로컬에 암호화 저장
- 네트워크 장애 시 데이터 유실 방지
- 운동 종료/자동 로그아웃 시 서버 전송 후 삭제

### 4.2 저장 데이터

```c
typedef struct {
    int user_id;
    int equipment_id;
    time_t start_time;
    double total_calories;
    int sensor_log_count;
    SensorData sensor_log[MAX_SENSOR_LOG]; // 최근 N개
} EncryptedSession;
```

### 4.3 라이프사이클

```
로그인 성공 → 로컬 저장소 초기화
  │
  ├─ 하트비트마다 → 센서 로그 추가, 칼로리 갱신
  │
  ├─ 운동 종료 → LOGOUT 메시지에 로컬 데이터 포함 전송
  │              → LOGOUT_OK 수신 후 로컬 삭제
  │
  └─ 자동 로그아웃 → FORCE_LOGOUT 수신
                    → 서버에 로컬 데이터 전송 시도
                    → 전송 성공/실패 무관하게 로컬 삭제
```

## 5. 메인 루프

```c
int main() {
    // 1. 설정 로드
    // 2. ncurses 초기화
    // 3. TCP 연결 + 기기 인증 (DEVICE_AUTH)
    // 4. 로그인 화면 표시

    while (running) {
        switch (current_state) {
            case STATE_LOGIN:
                handle_login_screen();
                break;
            case STATE_REGISTER:
                handle_register_screen();
                break;
            case STATE_DASHBOARD:
                // 3초마다 하트비트 전송
                // 서버 응답으로 대시보드 갱신
                // 사용자 입력 감시 (운동 종료)
                handle_dashboard();
                break;
            case STATE_WARNING:
                // 경고 오버레이 표시 + 카운트다운
                handle_warning_overlay();
                break;
        }
    }

    // 자원 정리
    cleanup();
    return 0;
}
```
