## 1. 테스트 전략

### 1.1 단위 테스트

| 대상 | 언어 | 도구 | 테스트 항목 |
|------|------|------|-------------|
| 칼로리 계산 | C | assert | 기구별 계산, 강도 계수 경계값 |
| SHA-256/bcrypt | C | assert | 해시 생성/검증 정확성 |
| 세션 판정 | C | assert | 타임아웃 판정 로직 |
| 프로토콜 파싱 | C | assert | JSON 빌드/파싱 정확성 |
| API 응답 | JS | Jest | 각 API route 응답 형식 검증 |
| 입력값 검증 | JS | Jest | zod 스키마 검증 |

### 1.2 통합 테스트

| 시나리오 | 방법 |
|----------|------|
| 기기→C서버→MySQL→daily_stats | 기기 시뮬레이터 기동 + DB 결과 확인 |
| 로그인→하트비트→로그아웃 전체 흐름 | 시뮬레이터 자동 시나리오 |
| 자동 로그아웃 5가지 시나리오 | 시뮬레이터로 각 상황 재현 |
| 웹 로그인→기록 조회→로그아웃 | Jest + Supertest |

### 1.3 보안 테스트

| 항목 | 방법 |
|------|------|
| SQL Injection | 비정상 입력 주입 |
| XSS | 스크립트 태그 입력 |
| CSRF | 크로스도메인 요청 |
| Rate Limit | 제한 초과 반복 요청 |
| 세션 탈취 | 만료/위조 세션 재사용 |

### 1.4 부하 테스트

| 시나리오 | 도구 | 기준 |
|----------|------|------|
| 50대 기기 동시 접속 | 다중 시뮬레이터 | 하트비트 응답 500ms 이내 |
| 웹 100명 동시 접속 | k6 / ab | API 응답 500ms 이내 |

## 2. 시연 시나리오

### 2.1 핵심 시연 흐름 (10분)

```
1. [Docker] MySQL + Redis 시작 (docker-compose up)
2. [C 서버] 서버 시작 → "서버 시작, 미완료 세션 정리" 로그
3. [기기 A] 기기 시뮬레이터 시작 → DEVICE_AUTH 성공
4. [기기 A] 전화번호 입력 → 미등록 → 등록 화면 → 등록 완료
5. [기기 A] 대시보드 표시: 칼로리 0, 시간 0
6. [기기 A] 센서 데이터 시작 → 칼로리/시간 실시간 증가
7. [웹] 기구 현황 페이지 → 기구 A "사용 중" 실시간 표시
8. [기기 A] 운동 종료 → 칼로리 정산 + 로그인 화면 복귀
9. [웹] 로그인 → 운동 기록 조회 → 차트 확인
10. [기기 B] 다른 기구 로그인 → 기기 A 아직 로그인 중이면 자동 로그아웃
11. [웹] 관리자 로그인 → 기구 관리 + 활성 세션 확인
```

### 2.2 자동 로그아웃 시연

```
1. 비활동 시나리오: 센서 IDLE 모드 → 2분 후 경고 → 3분 후 자동 로그아웃
2. 하트비트 시나리오: 네트워크 차단 시뮬 → 10초 후 자동 로그아웃
3. 중복 로그인: 기기 A 로그인 상태에서 기기 B 같은 사용자 로그인
```

### 2.3 보안 시연

```
1. 전화번호 마스킹: 웹에서 010-****-1234 표시
2. Rate Limit: 로그인 5회 초과 → 429 에러
3. 세션 관리: 로그아웃 후 쿠키 재사용 → 401
```

## 3. 시연 환경 준비

### 3.1 필요 환경

| 항목 | 스펙 |
|------|------|
| Docker | MySQL 8.x + Redis 7.x |
| C 컴파일러 | GCC (Linux) or MinGW/WSL (Windows) |
| Node.js | 24.14.1 |
| 터미널 | 4개 이상 (서버, 기기×2, 웹) |

### 3.2 시작 순서

```bash
# 1. 인프라
docker-compose up -d

# 2. DB 초기화
mysql -u root -p < sql/schema.sql
mysql -u root -p < sql/indexes.sql
mysql -u root -p < sql/triggers.sql
mysql -u root -p < sql/init.sql

# 3. C 서버
cd server && make && ./server

# 4. Next.js
cd web && npm install && npm run dev

# 5. IoT 기기 시뮬레이터
cd device && make && ./device --id=1
cd device && make && ./device --id=2  # 두 번째 기기
```
